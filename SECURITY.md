# Security policy

AppName is proprietary (see `LICENSE`). This file is the **vulnerability-reporting policy only**. Engineering rules for secrets, privacy, and the scanning guards live in [`docs/security.md`](docs/security.md).

## Reporting a vulnerability

Report suspected vulnerabilities privately using GitHub's private vulnerability reporting (the repository's **Security** tab, then **Report a vulnerability**). Do not open public issues for security problems, and do not disclose details publicly until a fix has been released.

Include a description and impact, steps to reproduce or a proof of concept, and the affected version, platform, and OS. You will receive an acknowledgement, and fixes are prioritized by severity.

## Supported versions

The project is pre-release; no versions are published yet. This section will list supported versions once releases begin.

## Scope

Reports about AppName's own code and configuration are in scope. Issues in third-party dependencies should be reported to their respective upstream projects.
